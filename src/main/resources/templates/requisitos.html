Implementação de JPA para persistência

Para tornar mais fácil e ágil o desenvolvimento de novas funcionalidades no sistema não precisando mais utilizar SQL nativo ao realizar transações com o banco de dados

Como um membro do time,

Preciso o projeto passe a utilizar JPA

Foi utilizado bibliotecas do próprio Spring para mapeamento e gerenciamento do banco de dados

Também foi utilizado para o Kotlin o plugin "no-arg" para enxugar as classes de mapeamento de dados, evitando, assim, de ter que criar getters e setters para todos os campos do Banco

Exemplo de classe:


data class Produto (
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    @Column(name = "codigo", nullable = false)
    var codigo: Int = 0,


    @Column(length = 200)
    @get: NotBlank
    var nome: String = "",

    @Column(length = 4000)
    @get: NotBlank
    var descricao: String = "",


    @ManyToOne(cascade = [CascadeType.MERGE])
    @JoinColumn(name = "categoria")
    @get: NotEmpty
    var categoria: Categoria? = null


)
Melhorias de design nas telas do sistema

Para melhorar a usabilidade

Como um usuário do sistema,

Preciso que sejam realizadas melhorias visuais nas telas do sistema.

Para melhorar a experiência de uso foi utilizado bibliotecas visuais do bootstrap.

Todo o visual foi cuidadosamente elaborado para ficar de fácil visualização tanto no computador quanto no celular.

Também para melhorar a usabilidade para internet lentas as bibliotecas CSS, JS e HTML são carregadas somente ao abrir a página pela primeira vez, posteriormente a alteração visual é toda feita por AJAX (Asynchronous Javascript and XML), permitindo pequenas "quedas" de conexão sem interferir a navegação além de diminuir a frequência de comunicação com o servidor e a quantidade de dados trafegados.
